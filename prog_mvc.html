Introducción

El patrón Modelo-Vista-Controlador (MVC) es una arquitectura de software que separa la lógica de la aplicación en tres componentes principales:

  • Modelo (Model): Representa los datos y la lógica de negocio.
  • Vista (View): Muestra la información al usuario.
  • Controlador (Controller): Maneja la interacción entre el modelo y la vista.

Ventajas

  • Separación de responsabilidades: Cada componente tiene una función específica, facilitando la organización del código.
  • Facilidad de mantenimiento: Separar la lógica de negocio, la presentación y la gestión de la interacción permite modificar o mejorar cada parte sin afectar a las demás.
  • Reutilización de código: La vista y el modelo pueden reutilizarse en diferentes partes del programa o incluso en otros proyectos.

Ventajas

  • Escalabilidad: Al estar bien estructurado, es más fácil añadir nuevas funcionalidades sin afectar el sistema en su conjunto.
  • Facilita la colaboración: Diferentes desarrolladores pueden trabajar simultáneamente en distintas partes del código sin interferencias.
  • Mejor prueba y depuración: Permite realizar pruebas unitarias más eficaces, ya que la lógica de negocio está separada de la interfaz gráfica.

Ejemplo básico con una clase base

4. Programa principal: Conectamos los componentes y probamos el patrón MVC.

						
							public class EstudianteApp {
								public static void main(String[] args) {
									// Crear modelo
									Estudiante modelo = new Estudiante("Juan Perez", 20);
									
									// Crear vista
									EstudianteVista vista = new EstudianteVista();
									
									// Crear controlador
									EstudianteControlador controlador = new EstudianteControlador(modelo, vista);
									
									// Actualizar vista inicial
									controlador.actualizarVista();
									
									// Modificar el modelo a través del controlador
									controlador.setNombreEstudiante("Ana Gómez");
									controlador.setEdadEstudiante(22);
									
									// Actualizar vista con nuevos datos
									controlador.actualizarVista();
								}
							}

Ejemplo básico con una clase base

4. Programa principal: Si necesitamos un menú e interactuar con el usuario pidiéndole datos, también iría en esta clase.

						
							public class EstudianteApp {
								public static void main(String[] args) {
									Scanner scanner = new Scanner(System.in);
							
									// Pedir datos iniciales del estudiante
									System.out.print("Introduce el nombre del estudiante: ");
									String nombre = scanner.nextLine();
									
									System.out.print("Introduce la edad del estudiante: ");
									int edad = Integer.parseInt(scanner.nextLine());
							
									// Crear modelo, vista y controlador
									Estudiante modelo = new Estudiante(nombre, edad);
									EstudianteVista vista = new EstudianteVista();
									EstudianteControlador controlador = new EstudianteControlador(modelo, vista);
							
									int opcion;
									do {
										System.out.println("\n--- MENÚ ---");
										System.out.println("1. Mostrar estudiante");
										System.out.println("2. Modificar nombre");
										System.out.println("3. Modificar edad");
										System.out.println("4. Salir");
										System.out.print("Seleccione una opción: ");
										opcion = Integer.parseInt(scanner.nextLine());
							
										switch (opcion) {
											case 1:
												controlador.actualizarVista();
												break;
											case 2:
												System.out.print("Nuevo nombre: ");
												String nuevoNombre = scanner.nextLine();
												controlador.setNombreEstudiante(nuevoNombre);
												break;
											case 3:
												System.out.print("Nueva edad: ");
												int nuevaEdad = Integer.parseInt(scanner.nextLine());
												controlador.setEdadEstudiante(nuevaEdad);
												break;
											case 4:
												System.out.println("Saliendo...");
												break;
											default:
												System.out.println("Opción no válida. Intente de nuevo.");
										}
									} while (opcion != 4);
								}
							}

Ejemplo completo con varias clases base

Vamos a implementar un ejemplo de MVC donde gestionamos la información de un estudiante que puede cursar muchas asignaturas.

Ejemplo completo con varias clases base

1. Modelo: Define los datos y la lógica de negocio.

						
							// Archivo Asignatura.java
							class Asignatura {
								private String nombre;
								private int creditos;
								
								public Asignatura(String nombre, int creditos) {
									this.nombre = nombre;
									this.creditos = creditos;
								}
								
								public String getNombre() {
									return nombre;
								}
								
								public int getCreditos() {
									return creditos;
								}
							}
							
							// Archivo Estudiante.java
							class Estudiante {
								private String nombre;
								private int edad;
								private Asignatura[] asignaturas;
								private int numAsignaturas;
								
								public Estudiante(String nombre, int edad, int maxAsignaturas) {
									this.nombre = nombre;
									this.edad = edad;
									this.asignaturas = new Asignatura[maxAsignaturas];
									this.numAsignaturas = 0;
								}
								
								public String getNombre() {
									return nombre;
								}
								
								public void setNombre(String nombre) {
									this.nombre = nombre;
								}
								
								public int getEdad() {
									return edad;
								}
								
								public void setEdad(int edad) {
									this.edad = edad;
								}
								
								public void agregarAsignatura(Asignatura asignatura) {
									if (numAsignaturas < asignaturas.length) {
										asignaturas[numAsignaturas++] = asignatura;
									}
								}
								
								public Asignatura[] getAsignaturas() {
									return asignaturas;
								}
								
								public int getNumAsignaturas() {
									return numAsignaturas;
								}
							}

Ejemplo completo con varias clases base

2. Vista: Muestra información al usuario.

						
							class EstudianteVista {
								public void mostrarEstudiante(String nombre, int edad, Asignatura[] asignaturas, int numAsignaturas) {
									System.out.println("Estudiante: ");
									System.out.println("Nombre: " + nombre);
									System.out.println("Edad: " + edad);
									System.out.println("Asignaturas: ");
									for (int i = 0; i < numAsignaturas; i++) {
										System.out.println("- " + asignaturas[i].getNombre() + " (" + asignaturas[i].getCreditos() + " créditos)");
									}
								}
							}

Ejemplo completo con varias clases base

3. Controlador: Gestiona la relación entre el modelo y vista.

						
							class EstudianteControlador {
								private Estudiante modelo;
								private EstudianteVista vista;
								
								public EstudianteControlador(Estudiante modelo, EstudianteVista vista) {
									this.modelo = modelo;
									this.vista = vista;
								}
								
								public void setNombreEstudiante(String nombre) {
									modelo.setNombre(nombre);
								}
								
								public String getNombreEstudiante() {
									return modelo.getNombre();
								}
								
								public void setEdadEstudiante(int edad) {
									modelo.setEdad(edad);
								}
								
								public int getEdadEstudiante() {
									return modelo.getEdad();
								}
								
								public void agregarAsignaturaEstudiante(Asignatura asignatura) {
									modelo.agregarAsignatura(asignatura);
								}
								
								public void actualizarVista() {
									vista.mostrarEstudiante(modelo.getNombre(), modelo.getEdad(), modelo.getAsignaturas(), modelo.getNumAsignaturas());
								}
							}

4. Programa principal: Conectamos los componentes y probamos el patrón MVC.

						
							public class EstudianteApp {
								public static void main(String[] args) {
									// Crear modelo (simulando su recuperación de BD)
									Estudiante modelo = getEstudianteFromDB();
									
									// Crear vista
									EstudianteVista vista = new EstudianteVista();
									
									// Crear controlador
									EstudianteControlador controlador = new EstudianteControlador(modelo, vista);

									controlador.setNombreEstudiante("Ana Gómez");
									controlador.setEdadEstudiante(22);
									controlador.agregarAsignaturaEstudiante(new Asignatura("Programación", 8));
									
									// Actualizar vista con nuevos datos
									controlador.actualizarVista();
								}

								private static Estudiante getEstudianteFromDB(){
									Estudiante estudiante = new Estudiante("Juan Perez", 20, 5);
									return estudiante;
								}
							}

Consideraciones finales

¿Realmente es práctico este patrón?

  • Los ejemplos proporcionados son muy sencillos con el objetivo de ilustrar el uso del patrón MVC, pero no son prácticos.
  • Para el modelo es más común tener una o varias clases que se inician sin datos, piensa en EstudiantesGestion o similar con un array de estudiantes (inicialmente con 0). En los siguientes ejercicios lo harás así.
  • Con una vista en consola estamos muy limitados, pero se le podrían añadir atributos para trabajar con diferentes colores, tema (claro u oscuro)...
  • Una práctica común es agrupar las clases en subpaquetes: modelo, vista o controlador.

Consideraciones finales

¿Y podrían crearse modelo y vista directamente en el controlador?

Sí, en aplicaciones pequeñas podrían instanciarse directamente dentro del controlador, siempre que no vayan a cambiar y usemos este patrón por motivos de organización.

Pero, en aplicaciones medianas o grandes, donde se necesita escalabilidad y facilidad de modificación, pasar el modelo y la vista ya creados al constructor del controlador, tiene muchas ventajas:

  • Mejora la flexibilidad al permitir reutilizar diferentes vistas y modelos.
  • Facilita la separación de responsabilidades evitando que el controlador instancie objetos.
  • Simplifica las pruebas unitarias al permitir inyectar dependencias de prueba (otros modelos específicos).
  • Reduce el acoplamiento, haciendo que el código sea más modular y mantenible.

Ejercicio 1: Gestor de tareas

Implementa un gestor de tareas donde los usuarios puedan agregar, listar y marcar tareas como completadas. Utiliza las siguientes clases:

Ejercicios

Ejercicio 1: Gestor de tareas

  • Modelo (Tarea.java): Atributos: id, descripcion, completada (booleano). Métodos: getters/setters, marcarComoCompletada().
  • Modelo (GestorTareas.java): Atributos: array de Tareas. Métodos: agregar tareas, buscarla, marcarla como completada...
  • Vista (GestorTareasVista.java): Métodos para mostrar una lista de tareas y mensajes al usuario. Atributos: emojiCompletada y emojiNoCompletada.
  • Controlador (GestorTareasControlador.java): Métodos para agregar una tarea, marcarla como completada y mostrar la lista.

Ejercicios

Ejercicio 1: Gestor de tareas

Ejemplo de uso:

						
							public class Main {
								public static void main(String[] args) {
									GestorTareas modelo = new GestorTareas();
									GestorTareasVista vista = new GestorTareasVista("✅", "❌");

									controlador.agregarTarea("Hacer ejercicio");
									controlador.mostrarTareas();
									controlador.marcarTareaComoCompletada(1);
									controlador.mostrarTareas();

							}
